# ruijie-campus-autologin

OpenWrt opkg package for a Ruijie/Dr.COM campus portal at `192.168.16.3`.
It runs as a `procd` service, checks login state periodically, and posts the
configured username, password, and operator suffix when the router is offline.

The demo page in this repository uses these login fields:

- `DDDDD`: username plus operator suffix
- `upass`: password
- `0MKKey`: submit key, default `123456`
- `R1`, `R2`, `R3`, `R6`, `para`: fixed compatibility fields

## Build

Copy `package/ruijie-campus-autologin` into an OpenWrt source tree or SDK, then:

```sh
make menuconfig
# Network -> ruijie-campus-autologin
make package/ruijie-campus-autologin/compile V=s
```

The generated `.ipk` will be under `bin/packages/.../base/`.

## Install

```sh
opkg install ruijie-campus-autologin_*.ipk
```

## Configure

Edit `/etc/config/ruijie-campus-autologin`:

```sh
uci set ruijie-campus-autologin.main.enabled='1'
uci set ruijie-campus-autologin.main.username='your_student_id'
uci set ruijie-campus-autologin.main.password='your_password'
uci set ruijie-campus-autologin.main.operator='cmcc'
uci commit ruijie-campus-autologin
```

Supported `operator` values:

- `cmcc` or `mobile`: append `@cmcc`
- `unicom` or `lt`: append `@unicom`
- `telecom` or `dx`: append `@dx`
- `none`, `campus`, `teacher`, `continuing`: append nothing
- any value starting with `@`: used as a literal suffix

If `username` already contains `@`, the script will not append another suffix.

## Start

```sh
/etc/init.d/ruijie-campus-autologin enable
/etc/init.d/ruijie-campus-autologin start
logread -f | grep ruijie-campus-autologin
```

## Login request generated by the service

For `username=your_student_id`, `password=secret`, and `operator=cmcc`, the service
posts roughly this curl request:

```sh
curl -fsSL \
  -A 'Mozilla/5.0 (OpenWrt; ruijie-campus-autologin)' \
  -e 'http://192.168.16.3/' \
  --data-urlencode 'DDDDD=your_student_id@cmcc' \
  --data-urlencode 'upass=secret' \
  --data-urlencode 'R1=0' \
  --data-urlencode 'R2=0' \
  --data-urlencode 'R3=0' \
  --data-urlencode 'R6=0' \
  --data-urlencode 'para=00' \
  --data-urlencode '0MKKey=123456' \
  'http://192.168.16.3/'
```

## Notes

The service considers the router logged in when the portal page contains the
logged-in Dr.COM marker `Dr.COMWebLoginID_1.htm` or the logout form markers. It
can also use an optional HTTP probe via `probe_url` and `probe_expect_code`.
